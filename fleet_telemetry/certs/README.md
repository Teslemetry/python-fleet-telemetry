# Vendored Tesla CA bundles

These PEM files are **Tesla's** Certificate Authority bundles, vendored verbatim
from the [`teslamotors/fleet-telemetry`](https://github.com/teslamotors/fleet-telemetry)
reference server (`config/files/`):

| File                 | Source (`config/files/`) | Purpose                     |
| -------------------- | ------------------------ | --------------------------- |
| `tesla_prod_ca.pem`  | `prod_ca.crt`            | Production vehicle client CA |
| `tesla_eng_ca.pem`   | `eng_ca.crt`             | Engineering/staging client CA |

They are the trust anchors used by the telemetry **server** to verify the client
certificates that Tesla vehicles present during the mTLS handshake (the
"server verifies the vehicle" direction). `fleet_telemetry.identity` then
re-checks the issuer Common Name against the set of recognized Tesla issuing CAs.

These are copied verbatim and must **not** be edited. To regenerate when Tesla
rotates its CAs, re-copy the upstream files:

```sh
cp path/to/fleet-telemetry/config/files/prod_ca.crt fleet_telemetry/certs/tesla_prod_ca.pem
cp path/to/fleet-telemetry/config/files/eng_ca.crt  fleet_telemetry/certs/tesla_eng_ca.pem
```

These bundles are **not** involved in the "vehicle verifies the server"
direction — that uses a root CA and server leaf generated by
`fleet_telemetry.certs.ServerCredentials`.
